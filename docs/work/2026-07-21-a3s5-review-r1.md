# A3-S5 freeze素案レビュー r1 (2026-07-21, Sol high) — NO-GO P0=4 P1=8 P2=2

総評: **NO-GO**。docs/12・docs/13 は witness Storage の正本を docs/17 に委譲しており、直接矛盾は見つかりません。一方、素案には successor transition を実装不能にする未確定点と、cycle bound の幅不足があります。
## P0
1. **ACTIVE 時 header digest の期待値が導出不能のまま**
   （docs/work/2026-07-21-a3-s5-freeze-draft.md:126,353）
   `primary_value_digest` は WITNESS_HEADER では常に zero なので導出元になりません。`record_revision` 単独からも complete `NLR1` value digest は得られません。このままでは §10.1 の old ACTIVE/new SUPERSEDED exact transition を証明できません。
2. **RE_SUPERSEDE 検出が存在しない wire field を前提としている**
   （docs/work/2026-07-21-a3-s5-freeze-draft.md:110,354）
   `SUPERSEDE_WITNESS` entry に「new successor field」はありません。存在するのは old/new value digest だけです。現在の記述は捏造規定であり、mandatory finding を構成できません。
3. **RETIRED successor で停止した後の chain authority が未割当**
   （docs/work/2026-07-21-a3-s5-freeze-draft.md:107,153-167,352）
   停止前に当該 hop の successor manifest を証明するのか不明で、停止後の RETIRED→successor suffix は §18.2 の現行 S6 行にも明示所有されていません。合法な `SUPERSEDED W1 → RETIRED W2` snapshot を complete/corrupt のどちらにも閉じられません。
4. **`walk_bound` / `walk_steps` の u16 は無根拠な切捨て**
   （docs/work/2026-07-21-a3-s5-freeze-draft.md:215-216,356）
   witness header 総数に 65535 上限はありません。u16 overflow/truncation により長い合法 chain または cycle を誤判定します。§18.5 category E の「witness row count」を保持できていません。
## P1
1. **Mode36 は D1/D2 所有の same-record check を D3-S5 に再発明している**
   （docs/work/2026-07-21-a3-s5-freeze-draft.md:39-46,134-145）
   ACTIVE successor-zero は docs/17 §10.1 の header-local invariant で、§18.6 も header local を D1/D2-S3 所有としています。S5 の data-dependent work は SUPERSEDED 起点の bounded walk だけなので、closed set は `k₅=1 / Mode35` が整合的です。
2. **same-txn phase machine が iterator と hop promotion を閉じていない**
   （docs/work/2026-07-21-a3-s5-freeze-draft.md:50-68,83-87,252）
   BASELINE 後の sole iterator close/reopen、`last_carrier_key` の保存、successor get 後に current/previous/successor digest をどの transition でshiftするかが未規定です。現状の field 名と「step 3 で advance」だけでは同時 pin の所有が一意になりません。
3. **manifest proof の exact stream 規則が不足**
   （docs/work/2026-07-21-a3-s5-freeze-draft.md:83-89,120-132）
   次が未固定です。
   - 非末尾8／末尾 exact entry count
   - 同一 chunk を ordinal ごとに再取得しても SHA へは `i%8==0` の一度だけ投入
   - target entry の二重出現拒否
   - `entry_found==1` を要求した full-M close
   §18.15.3 と同等の厳密度に達していません。
4. **finalize の required-mask/result-carrier が exact shape まで閉じていない**
   （docs/work/2026-07-21-a3-s5-freeze-draft.md:149-178,244-252）
   Mode別 required exact masks、empty inventory時のbit設定、invalid extra-bit shapeが明文化されていません。また S5 carrier の declaration offset、host/target sizeof、`<=64` assert、finalize stack accountingが欠落しています。
5. **quadratic cost の manifest 項を過少計上**
   （docs/work/2026-07-21-a3-s5-freeze-draft.md:184-187）
   long chain では manifest cost は `Θ(Σ_j j·M_j)`、一様なら `Θ(L²·M_avg)` です。記載の `Θ(L² + L·M_avg)` は各 manifest が複数 origin から再走査される事実を落としています。
6. **S1+S2+S5 ceiling が同一行で矛盾**
   （docs/work/2026-07-21-a3-s5-freeze-draft.md:263）
   `9152 unchanged` は誤りで、`8384+448+320+512=9664` が正です。full outer `11392` と packed `11311→align8 11312` は現行497-byte案なら算術上正しいです。
7. **mandatory mutation catalog が中核 adjudication を覆っていない**
   （docs/work/2026-07-21-a3-s5-freeze-draft.md:283-314）
   RETIRED handoff、revision≠2、合成ACTIVE digest不一致、target entry duplicate、wrong final entry-count、tail-into-cycle、successor state invalid、SHA duplicate-feed が必須ケースにありません。
8. **digest collision を D1で検出済みとする主張が不正確**
   （docs/work/2026-07-21-a3-s5-freeze-draft.md:114）
   D1 が証明できるのは「返されたkey/body rawが同じ composite digestを再計算する」ことまでです。SHA-256 collision が本当に発生した場合、digestだけを保持する predecessor から意図した別raw identityを復元・識別できません。「D1で検出済み」ではなく、collision-resistance assumption と raw/key bijection を分けて記述すべきです。
## P2
1. **`ACTIVE_REGRESSION` が `MISSING_ENTRY` と同一条件**
   （docs/work/2026-07-21-a3-s5-freeze-draft.md:109,160）
   ACTIVE terminus 自体は正常です。manifest entry 不在は state に関係なく `MISSING_ENTRY` で十分であり、別 finding 名は推測分類です。
2. **align 1 の object に「align padding」として reserved を置いている**
   （docs/work/2026-07-21-a3-s5-freeze-draft.md:230,357）
   全fieldをbyte arrayで表すなら自然 alignment は1で paddingはありません。末尾3 bytesは単なる reserved MBZです。

- modes は **Mode35のみ、k₅=1**。Mode36を削除し、S5 contextでは36..255 invalid。
- u64化後の context は、他fieldを維持すれば **sizeof 509 / align 1 / ceiling 512**。
- packed full は `8384+421+306+754+949+509=11323`、align8 **11328**、outer ceiling **11392** は維持可能。

- S5 result bytesはS4 bytesの後、host declaration offset **54/55**。現行host `sizeof=56` を維持し、target `sizeof<=64` をassertする。
