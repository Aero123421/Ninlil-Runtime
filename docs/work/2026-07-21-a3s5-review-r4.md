## P1
1. **`prev_hop_witness_digest`のpromotion代入元が誤り**
   現在は`prev_hop := hop_witness_digest`、`current := hop_witness_digest`となり、両方が新currentになります。[freeze draft:111](/Users/dt/job/LoRa/ninlil-fable-a3-s5/docs/work/2026-07-21-a3-s5-freeze-draft.md:111)
   正しくはcurrentを上書きする前に、旧`witness_digest`をpreviousへ保存する必要があります。
   ```text
   prev_hop_witness_digest := witness_digest
   witness_digest := hop_witness_digest
   ```
   現状ではW1→W2→W3→W2のtail bounceで、W3 promotion後の`prev_hop`がW3となり、W2との即時bounce比較が成立しません。bounded walkでは最終的に捕捉できますが、明記されたimmediate-bounce規則とexact state traceに反します。[freeze draft:162](/Users/dt/job/LoRa/ninlil-fable-a3-s5/docs/work/2026-07-21-a3-s5-freeze-draft.md:162)
2. **positive #1の`W2 manifest M=1`は構成不能**
   `SUPERSEDE_WITNESS(W1)`は、W2がW1をheadとするsemantic memberを更新した結果として追加されます。したがってW2 manifestには最低でもsemantic member 1件とW1 header entry 1件が必要で、`M≥2`です。[main:1108](/Users/dt/job/LoRa/ninlil-fable-a3-s5/docs/17-foundation-domain-store.md:1108)
   `M=1`でaction 4だけのpositiveはcanonical builderから生成できません。[freeze draft:415](/Users/dt/job/LoRa/ninlil-fable-a3-s5/docs/work/2026-07-21-a3-s5-freeze-draft.md:415) `M=2`へ直し、ordinary semantic entryを含めれば構成可能です。
3. **worked exampleがW2 carrierを処理せず早期COMPLETEしている**
   W2自身もSUPERSEDEDなので、W1 originのwalk終了後、SELECTはW2を独立carrierとして再選択しなければなりません。「no more SUPERSEDED carriers」は誤りです。[freeze draft:580](/Users/dt/job/LoRa/ninlil-fable-a3-s5/docs/work/2026-07-21-a3-s5-freeze-draft.md:580)
   M=2の例では完全なget列は次です。
   - W1 origin: `H(W2), C(W2,0)×2, H(W3), C(W3,0)×2`
   - W2 origin: `H(W3), C(W3,0)×2`
   - その後にのみCOMPLETE
   よってpositive #2/#4のW1-origin部分は構成可能ですが、記載されたsession全体のtraceは未完です。
positive #5は構成可能です。W1→W2 RETIREDで`hop_witness_digest=W2`を使ってW2 manifestをfull-M検証し、その完了後にbit5を設定してS6へhandoffできます。
