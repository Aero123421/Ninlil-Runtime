## context lifecycle 遷移表（P0-2）

| 現状態 | イベント | 次状態 | wire / install | 備考 |
|---|---|---|---|---|
| `NONE` | `install_from_token` / `install_responder` 成功 | `ACTIVE` | seal/open 可 | `freshness_epoch++`、Hop TX/RX + E2E TX/RX 導出、counter=1 から burn |
| `ACTIVE` | `wire_seal_*` / `wire_open_*` | `ACTIVE` | 継続 | counter 単調増加、nonce 一意、AEAD + CRC 境界 |
| `ACTIVE` | `rx_precheck` 重複 / 逆行 counter | — | reject `REPLAY` / `COUNTER` | false success 0 |
| `ACTIVE` | AEAD/CRC/構造改竄 | — | reject `AUTH_FAILED` / `STRUCTURAL` | false success 0 |
| `ACTIVE` | `context_clean_restart` | `RESTART_REQUIRED` | `FENCED` | hop/e2e key zero、旧 epoch fence |
| `ACTIVE` | `context_commit_unknown_restart` | `COMMIT_UNKNOWN` | `FENCED` | 同上 |
| `RESTART_REQUIRED` | wire 操作 | — | `FENCED` | fresh handshake 必須 |
| `RESTART_REQUIRED` | 新 token で install 成功 | `ACTIVE` | 再開 | `next_session_id` / `next_hop_context_id` 進行で新 token |
| `RESTART_REQUIRED` | 消費済み / 同一 fingerprint token 再注入 | — | `STALE` | ALL_PROPOSED 後の再注入禁止 |
| `COMMIT_UNKNOWN` | install / wire | — | `FENCED` | M5 なし；fresh M4 へ戻る前は操作不可 |
| `ACTIVE` | 期限切れ install token consume | — | `STALE` | T1c 検証失敗は明示 reject |

Hop/E2E 各方向 × DATA/ACK lane を store 内に保持。restart 後 `nonce_reuse_count == 0`（E2E 証拠）。

## T1c token consume → authenticated install

| 側 | API | 方向 |
|---|---|---|
| controller | `ninlil_c3_lab_install_from_token` | Hop TX IR / Hop RX RI、E2E TX/RX 導出 |
| endpoint | `ninlil_c3_lab_install_responder` | Hop TX RI / Hop RX IR、E2E TX/RX 導出 |

traffic secret = M4 と同式 HKDF（proof_hmac + challenge_nonce + session_id）。token 検証失敗・stale・fingerprint 再注入は `STALE` / `FENCED` / `REJECTED`（false success 0）。

## counter / nonce / AEAD / replay / durable admission

| 機構 | 実装 | 負例 |
|---|---|---|
| counter burn | `ninlil_c3_lab_tx_burn_counter`（送信前単調増加） | 逆行受信 → `COUNTER` |
| nonce | counter + lane + layer から導出（reuse 検出 `nonce_reuse_count`） | 再利用 0（E2E restart 後確認） |
| AEAD | `r7_wire_codec` seal/open、CRC32C 認証境界 | tamper → `AUTH_FAILED` |
| replay window | 64-bit bitmap、`rx_precheck` → open → `rx_admit` | 同一 frame 再送 → `REPLAY` |
| durable admission | key `C3R` 16B + value 48B（CRC @ offset 44） | writer gate kind 32 / op 16 |

## 両 endpoint restart E2E（`c3_lab_secure_1hop_e2e_test`）

単一プロセスで controller ↔ endpoint 双方向交換:

1. M4 handshake → C3 install（両側）
2. controller → endpoint DATA（counter 1）、duplicate → `REPLAY`（`dup_reject == 1`）
3. endpoint → controller DATA（逆方向 counter 継続）
4. 両側 `context_clean_restart` → seal `FENCED`
5. `next_session_id` / `next_hop_context_id` 進行 → fresh handshake → 再交換
6. 重複 0（`dup_reject == 0`）、counter 1 から再開、`nonce_reuse_count == 0`
7. `context_commit_unknown_restart` → `COMMIT_UNKNOWN`
